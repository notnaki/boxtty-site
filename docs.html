Documentation

boxtty is a native SwiftUI Gmail client for macOS — keyboard-first, local-first, and scriptable. This covers setup through automation.

Overview

boxtty talks to Gmail's REST API directly and keeps your working set in memory, so the UI is instant. Every mutation — archive, star, trash, snooze, send — is optimistic, with an undo toast, and reconciles against the server in the background. New mail is pushed over IMAP IDLE rather than polled.

All state lives under ~/.config/boxtty as plain JSON and TOML. Nothing is sent anywhere except Gmail and, if you opt in, the AI provider you choose. The default AI runs on-device.

Requirements: macOS 14+. Apple Intelligence features need macOS 26+ with Apple Intelligence enabled.

Install & build

boxtty is in open beta and built from source. You'll need Xcode and XcodeGen (brew install xcodegen).

git clone https://github.com/notnaki/boxtty
cd boxtty/boxtty-mac
./build.sh                 # → dist/Boxtty.app and dist/boxtty
open dist/Boxtty.app

Or from Xcode:

xcodegen generate
open Boxtty.xcodeproj       # ⌘R

On first launch with no account connected, boxtty runs against in-memory mock data so you can try the UI immediately.

Connect Gmail

boxtty uses your own Google OAuth Desktop client — no shared app, no middleman. Create one in the Google Cloud console, then drop the credentials in an untracked file:

# ~/.config/boxtty/oauth.toml
client_id = "…apps.googleusercontent.com"
client_secret = "…"

The OAuth client must list http://127.0.0.1:8000 as an authorized redirect URI and request the https://mail.google.com/ scope.

Then in the app: ⌘K → connect (or the sidebar's add account) → sign in with Google. boxtty opens your browser, runs the auth-code + PKCE flow against a one-shot loopback listener on 127.0.0.1:8000, and stores the token in ~/.config/boxtty/tokens-<email>.json. No password ever touches the app. It auto-connects on the next launch.

Multiple accounts

Run connect again to add another Gmail account. boxtty merges them into one unified inbox: each row carries a per-account colour stripe, each account gets its own push connection, and replies are sent from the account that owns the thread automatically.

Manage accounts from the connect sheet — remove one with the ✕ next to it. Tokens are stored per account; an upgrade from a single-account install migrates automatically.

Search

The search field combines free text with operators:

OperatorMatches
from:Sender name or address
is:unreadUnread threads
is:readRead threads
is:starredStarred threads

Combine freely with free text, e.g. from:linear is:unread deploy. Searching reaches the server, so older matches outside the synced window are paged in.

Compose

Open the composer with c, ⌘N, or a reply / forward. It includes:

  • Recipient autocomplete — a dropdown of everyone you've corresponded with, matched on name or address, with already-entered recipients filtered out. Pick with ↑↓ + return/tab, or click.
  • Shorthand snippets — one-click sign-offs and openers above the body.
  • Markdown shortcuts — **bold**, _italic_, [text](url), and backtick code compile to HTML on send (toggle in settings).
  • Undo send — a window (configurable, off by default) where the message can be pulled back before it leaves.
  • Schedule send — the clock menu sends later at a chosen time.

Sent mail lands in Sent; replies thread correctly and quote the prior message collapsed.

Snooze & schedule

Press z to snooze a thread out of the inbox until later — it moves to the Snoozed folder and pops back when due. Scheduled sends and snoozes are stored locally and survive relaunch; due items fire whenever the app is running.

Filters

Rules run automatically against newly-arrived threads. Each rule matches on sender or subject (contains / does-not-contain / is) and applies an action: archive, trash, mark read, or star. Manage them under Settings → Filters; run them across the current inbox on demand.

CLI

While the app is running it hosts a Unix-socket control server at ~/.config/boxtty/control.sock. The boxtty tool is a thin client:

boxtty               # no args → opens the GUI
boxtty commands      # full verb list
boxtty list          # current folder as a table
boxtty unread        # inbox unread count
boxtty archive 102   # archive / trash / star / read / snooze <id>
boxtty select starred
boxtty send --to a@b.com --subject hi --body "from the cli"

Lua scripting

boxtty embeds Lua 5.4. boxtty run [script.lua] (default ~/.config/boxtty/init.lua) runs your script with a boxtty table that drives the app over the same socket:

for _, t in ipairs(boxtty.list().threads) do
  if t.from == "GitHub" then boxtty.archive(t.id) end
end

print("unread: " .. boxtty.unread())

boxtty.send{ to = "a@b.com", subject = "hi", body = "sent from lua" }

The CLI verbs and Lua functions are the same surface: list · unread · count · folders · selected · open · archive · trash · star · read · snooze · unsnooze · reply · forward · select · search · reload · compose · send. See examples/init.lua in the repo.

Event hooks

Drop a script at ~/.config/boxtty/hooks.lua and boxtty runs it whenever something happens — currently message_received on new mail. The event is passed in the environment and the script acts through the same boxtty.* API:

-- ~/.config/boxtty/hooks.lua
if os.getenv("BOXTTY_EVENT") == "message_received" then
  local id   = tonumber(os.getenv("BOXTTY_THREAD_ID"))
  local from = os.getenv("BOXTTY_FROM")
  if from:find("GitHub") then boxtty.archive(id) end
end

Variables: BOXTTY_EVENT, BOXTTY_THREAD_ID, BOXTTY_FROM, BOXTTY_FROM_EMAIL, BOXTTY_SUBJECT, BOXTTY_ACCOUNT. See examples/hooks.lua.

Troubleshooting

Auth fails or mail won't load

Confirm oauth.toml has http://127.0.0.1:8000 as a redirect URI and the https://mail.google.com/ scope. Re-run connect from ⌘K.

New mail isn't instant

Push uses IMAP IDLE; if your network blocks it boxtty falls back to periodic sync. Reads still reconcile on reload.

Self-check

Verify the core logic asserts on your build:

dist/Boxtty.app/Contents/MacOS/Boxtty --selfcheck   # exits 0 / 1

More questions? Open an issue on GitHub.
